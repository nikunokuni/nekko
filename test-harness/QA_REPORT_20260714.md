## A. バグ（動作が壊れる／データが壊れる）

1. **KIF棋譜の「右／左／直／上／引／寄」付きの手で読み込みが打ち切られる**
   `kifuParser.js:199-208` — 移動元 `(49)` を取り除いた後の駒名が「金右」「銀直」等になると
   `KIF_PIECE` に一致せず null → その手以降すべて破棄。
   実行検証: `５八金右(49)` を含む6手の棋譜 → 4手分しか読めず skipped=2。
   Kifu for Windows・ぴよ将棋等のKIFは相対指定を普通に含むため、実戦譜の多くが途中までしか読めない。
   移動元座標があれば相対指定は冗長情報なので、駒名の末尾から `[右左直上引寄行]` を捨てるだけで直る。

2. **「変化：」を含むKIFで分岐の手が本譜に連結され、局面が壊れる**
   `kifuParser.js:222-240` — 変化ヘッダ行は無視されるが、その下の手番号付きの行は
   `^(\d+)\s+` にマッチして本譜の続きとして apply される（`applyMove` は盤面整合を検証しない）。
   実行検証: 本譜4手+変化2手のKIF → snapshots が7枚になり、5枚目以降は不正な局面。
   検討用に変化を保存したKIFを読み込むと、エラーも出ずに壊れた棋譜ができる。
   対策: 「変化：」行以降を読み捨てる（または手番号が巻き戻ったら打ち切る）。

3. **盤面エディタで玉を取ると駒が消滅する（見えない持ち駒に入る）**
   `ShogiBoard.jsx:447-450` — 駒取り時に `baseKey(target)` を無条件で持ち駒へ加算するが、
   持ち駒UI（`HAND_ORDER`, `ShogiBoard.jsx:114`）に `k` が無いため、取られた玉はどこにも表示されず
   盤へ戻す手段がない。`hand_sente` に `k:1` が残った状態でDBにも保存される。
   `depositToHand`（`ShogiBoard.jsx:478`）には玉ガードがあるのに、駒取り経路には無い非対称。
   対策: 玉を取ったら持ち駒に入れない（消すか、取り自体を禁止）。

4. **公開／公開取り消しの失敗が完全に無言（モーダルのエラーUIは到達不能）**
   `App.jsx:411-432` — `handlePublishTree` / `handleUnpublishTree` が例外を catch して console に
   出すだけで再スローしないため、`EditTreeModal`（`TreeListScreen.jsx:138-166`）の
   `catch → setPublishError(...)` は決して実行されない死にコード。
   失敗時もモーダルは成功時と同じ挙動で閉じ、カードのバッジも変わらないので、
   ユーザーは「公開できたのか」判断できない。
   対策: App側で再スローする（onSaveと同様）か、App側で alert する。

5. **盤面キャンバス上を通るスクロールで駒が選択・移動される**
   `ShogiBoard.jsx:491-497` — `onTouchEnd` に移動量の判定がなく、盤の上で指を離すと
   常に `handleClick` 扱いになる。ノード詳細を1本指でスクロールして盤上で指を離すと
   駒が選択され、次のスクロールで意図しない移動・駒取りが起きうる。
   対策: touchstart 位置を記録し、一定距離（例: 10px）動いていたらタップ扱いしない。

6. **同じ棋譜ファイルをもう一度選んでも読み込まれない**
   `TreeListScreen.jsx:38-59` — `<input type="file">` の value をリセットしていないため、
   「読み取れませんでした」→ ファイルを修正して同名ファイルを再選択、の流れで change が発火しない。
   対策: `handleFileChange` の先頭または末尾で `e.target.value = ""`。

7. **失敗が無言のまま終わる操作が他にも数カ所**
   - ツリー新規作成の失敗（`App.jsx:334,349` console.error のみ。タップしても何も起きないように見える）
   - ノード削除の失敗（`App.jsx:655-657` console.error のみ。確認UIだけ閉じて削除されない）
   - 分岐追加・ノード追加の失敗（`App.jsx:562,586` `if (!newNode) return;` で無反応）
   他の操作（保存・移動・コピー等）は alert を出しており、基準が揃っていない。

## B. 表示と機能が違う

8. **「親ノードの盤面を引き継いでいます」バナーが実態と無関係に出続ける**
   `uiParts.jsx:205-221` — 表示条件が「親に盤面があること」だけなので、テンプレートを読み込んでも、
   駒を全部並べ替えても、棋譜途中分岐で作ったノードでも出続ける。しかもルートノードは
   常に盤面を持つ（`App.jsx:323,343`）ため、実質「盤面を出した全ノード」に常時表示される。
   対策: 盤面追加直後（引き継ぎした編集セッション中）だけ表示する、または文言を
   「親ノードにも盤面があります」等の事実ベースに変える。

9. **「棋譜を削除」が盤面まで記録開始時点に巻き戻す**
   `NodeDetailScreen.jsx:653-665` — kifu を消すだけでなく board / 持ち駒を `kifu[0]` に戻す。
   棋譜記録後に盤面を編集していた場合、その編集も消える。ボタン名は「削除」なのに
   アイコンは ti-arrow-back-up（戻す矢印）で、どちらの動作なのか外観から判断できない。
   title属性の「入力前の盤面に戻す」はモバイルでは見えない。
   対策: 「棋譜を削除（現在の盤面は維持）」と「記録開始時点に戻す」を分ける、
   少なくともラベル・アイコンを実挙動に合わせる。

10. **ステータス「未定」が存在するのに選べない・戻せない**
    凡例（`MindMapScreen.jsx:746-754`）・一覧・目次には「未定」があり、ルート直下の自動生成
    ノードも todo で作られるが、ノード詳細のチップは `["done","wip"]` のみ
    （`NodeDetailScreen.jsx:560`）。一度「完成/研究中」にすると未定へ戻せない。
    また手動追加ノードは wip、自動生成ノードは todo と初期値も不統一（`App.jsx:559` vs `db.js:120`）。

11. **トロフィーの達成判定基準が2系統混在し、表示が実績とズレる**
    - アクション系（いいね・コピー・棋譜記録・テンプレ・カスタムタグ）と累計/連続ログインは
      localStorage（`rewards.js`）依存 → 別端末・別ブラウザ・ストレージ削除で未獲得表示に戻る。
    - 公開・メモ・タグ系は現在の `myTrees` から算出（`App.jsx:835-843`）→ 公開を取り消したり
      ツリーを消すと獲得済みバッジが消える。
    「一度獲得したら残る」のか「現在の状態を表す」のか、どちらにも読める表示になっている。

12. **新規登録フォームの表記と検証が一致しない**
    `screensPublic.jsx:174` — placeholder は「8文字以上」だがクライアント検証はなく、
    Supabase 既定（6文字）だと7文字でも通る。逆に短すぎる場合のエラーは
    "Password should be at least 6 characters" 等の英語がそのまま表示される
    （`screensPublic.jsx:95` err.message 直出し。ID重複時の "User already registered" も同様）。
    また password の autocomplete が signup 時も "current-password"（`screensPublic.jsx:42`）で、
    パスワードマネージャが新規生成を提案しない。

## C. 直感的でない

13. **棋譜があると「棋譜を記録」ボタンが説明なく消える**
    `ShogiBoard.jsx:538` — 上書き防止のため非表示にする設計だが、「削除すれば再記録できる」
    ことがUI上どこにも示されない。ボタンを disabled + 理由表示にする方が伝わる。

14. **オンボーディングのトーストが4.5秒で自動送りされ、未読でも既読になる**
    `App.jsx:203-211` — 複数枚のトーストはタップしなくても自動で進み、最後まで進むと
    `markOnboardSeen` される。画面を見ていなかっただけで永久に消える（設定から再表示は
    可能だが、その導線に気づく頃には手遅れ）。自動送りは「進める」だけにして、
    既読化はタップ操作時に限定する方が安全。

15. **「ついか」見出しの金色「・」の意味が分からない**
    `NodeDetailScreen.jsx:677` — 未入力項目（志向・勝率）があると付くインジケータだが、
    凡例も tooltip もなく、汚れのようにも見える。「未入力あり」等のテキストが明快。

16. **タグピッカーのグループチップ本体タップが今も「タグ選択＋一覧開閉」を兼ねる**
    `uiParts.jsx:320-323` — 前回QA(#9)で丸い開閉ボタンは追加されたが、チップ本体を
    タップすると相変わらずグループ名タグの付け外しと開閉が同時に起きる。
    一覧を見たいだけの人が本体をタップする誤操作は残っている。

17. **みんなのツリー：自分のツリーにもいいね・コピーができる／作者名で検索できない**
    `screensPublic.jsx:324-327` — 検索対象は名前とタグのみ。自分のツリーも一覧に出て、
    自分でいいねを付けてランキング（liked_by降順）を上げられる。

18. **盤面がスマホでぼやける**
    `ShogiBoard.jsx:602` — canvas が 342×342 の論理ピクセルのまま CSS 拡大されるだけで、
    devicePixelRatio を掛けていないため、Retina端末では駒文字がにじむ。

19. **トロフィーのノード数が「作った」実感とズレる**
    `db.js:226-235` — ルートは除外するが、ツリー作成時に自動生成される「居飛車/振り飛車」
    （ツリーごとに+2）や、コピーで取り込んだ全ノードもカウントされる。
    「ノードを10個作る」はツリーを5個作るだけで達成できる。

20. **細かい点**
    - `App.jsx:436` の `nodeId === "new"` 分岐はどこからも呼ばれない死にコード。
    - `approach_type` カラム（`db.js:119` ほか）はUIに設定手段がなく常に null の死にデータ。
    - 金曜夜トーストの出題対象（`db.js:108`）に休止中ツリーのノードも含まれる。
    - ツリー編集モーダルは開いたまま名前を保存すると props が古いまま残り、同名でも
      blur のたびに再保存される（`TreeListScreen.jsx:125-129`、実害なし）。
    - みんなのツリーの「更新」ボタンに読み込み中の表示がない。
