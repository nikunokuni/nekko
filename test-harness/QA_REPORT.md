# 手動QAレポート — 居飛車ツリー構築テスト（2026-07-13）

## テスト方法

- 実バックエンド（Supabase）の資格情報がないため、`@supabase/supabase-js` を
  `test-harness/supabaseMock.js`（localStorage 永続のインメモリ実装）に差し替えて実行。
  アプリ本体のコードは無改変。起動: `npx vite --config vite.mock.config.js`
- Playwright（Chromium・390×844 のモバイル相当ビューポート）で全操作をUIから実施。
- ユーザー `niku` で新規登録し、居飛車党視点の研究ツリー「居飛車」を構築。
  最終状態: **非ルートノード70個（ルート含め71個）**、公開済み・いいね1・一言メモあり。

## 発見したバグ

1. **盤面の「非表示」が保存されない** — 非表示にしてマップへ戻り、ノードを開き直すと盤面が再表示される（`handleToggleBoard` がDB更新しないため）。
2. **盤面を「追加」しただけでは保存されない** — 追加（初期配置 or 親盤面の引き継ぎ）後、駒を動かす等の編集をせずに画面を離れると盤面が消える。「盤面を追加してメモだけ書いた」つもりのユーザーはデータを失う。
3. **ルート直下に追加したノードの並び順がリロードで変わる** — 新規ノードの `sort_order` が常に0のため、追加直後は末尾（振り飛車の後）だが、ツリーを開き直すと「居飛車 › 新ノード › 振り飛車」の順に割り込む。実測で確認。
4. **マップのドラッグ中にノードのラベル文字が範囲選択される**（SVGテキストに `user-select: none` がなく、付け替えドラッグの後にラベルが青く選択されたまま残る）。
5. **ホイールズームで毎回コンソールエラー** — `Unable to preventDefault inside passive event listener`（React の `onWheel` は passive）。また判定が `Math.abs(e.deltaY) < 1` なので、修飾キーなしの普通のスクロール操作もすべてズームになる。

## 気になる仕様・UX

6. **70ノード規模ではマップ初期表示にルートが映らない** — 初期表示・「表示リセット」ともキャンバス左上(20,20)固定で、大きいツリーでは最深部の葉だけが見える。ズーム下限0.4でも全体（横幅5000px超）が画面に入らず、全体俯瞰の手段がない。
7. **目次（ドロワー）からルートノードへ移動できない**（目次はルートの子から始まる）。ルートを見失うと戻る手段がパン操作しかない。
8. **合流線は常にツリー全体の右端まで迂回する** — 隣接ノード同士の合流でも画面外まで往復する長い点線になり、合流元→先を目で追えない。
9. **タグピッカーのグループ名タップが「開閉」と「タグ選択」を兼ねる**（居飛車/振り飛車グループ）— 別グループを覗くだけでそのグループ名タグが付き、閉じるともう一度トグルされる。誤操作しやすい。
10. **パンくずが「居飛車 › 居飛車 › …」** — ツリー名とルートノード名が同じため二重に見える。
11. **一言メモはツリー一覧カードとマップのドロワーで同一フィールド** — 片方で書くともう片方が上書きされる（同期される仕様だが、別メモだと思って使うと消える）。
12. **棋譜記録中に1手だけ戻す手段がない** — 間違えたら記録し直し。また盤面エディタとして自分の駒で自分の駒を取れる（自由編集としては妥当だが、記録中の誤タップに気づきにくい）。
13. **盤面の「元に戻す」の基準は“この画面を開いた時点”** — 一度ノードを離れて戻ると基準が更新され、直前の編集ミスを取り消せない。
14. **トロフィー画面の紙吹雪が開くたびに毎回出る**（新規獲得時に限らない）。
15. **オンボーディングの盤面用トースト（8枚）が盤面の真上に重なる** — タップが盤操作ではなくトースト送りに吸われる。また途中で画面を離れると既読にならず、次回また1枚目から全部出る。
16. **駒を1手動かすたびに board 全体を即UPDATE** — デバウンスがなく、駒を並べ直すと書き込みが連発する（モバイル回線・Supabase無料枠だと気になる量）。
17. **アイコンがCDNのWebフォント頼み** — CDN不達環境（今回のサンドボックス）ではヘッダー等のアイコンボタンが無地になり、パディングのないボタン（ドロワーの×など）は実質押せなくなる。PWAマニフェストを持つのにアイコンはオフライン非対応。
18. `apple-mobile-web-app-capable` の deprecated 警告が出る（`mobile-web-app-capable` 併記推奨）。
19. 細かい点: 成長アイコンはノード数×3px（最大300px）で、70ノードでは巨大な🌳がルート上に表示される（愛嬌の範囲）。公開/取り消しボタンは押すとモーダルが即閉じ、成功の明示はカードのバッジ変化のみ。

## 良かった点（参考）

- 合流・親/子付け替えの循環防止（子孫を候補から除外）が正しく機能。
- ノード削除時に他ノードの `merge_parent_ids` から参照を掃除するのを確認。
- コピー（copy_tree 相当）で合流リンク・親子関係がIDリマップ付きで維持される。
- ラベル等のデバウンス保存 + blur/visibilitychange でのflushは堅実。
- 棋譜と盤面が食い違ったときの「別に編集されています」表示は親切。
