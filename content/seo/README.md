# ねっこ SEO記事セット｜30キーワード × 短編記事

戦法メモアプリ **ねっこ**（https://nekko-weld.vercel.app ）への検索流入をつくるための、SEO記事の下書き集です。
現状アプリは認証裏のSPAで indexable なコンテンツがほぼないため、**外部（note／個人ブログ）に記事を出して流入を作る**ことを狙っています。

- 記事はすべて `articles/` 配下の Markdown（note／ブログにそのまま貼れます）
- **1キーワード＝1本**の短編（約600〜1,000字）
- トーン：ねっこのブランドに合わせた、穏やか・共感的・成長メタファー（根／芽／木）

## 導線：ハブ&スポーク構成

この30記事は集客の入口（スポーク）で、既存の **note使い方記事をハブ** に据えて次の流れに集約します。

```
悩みキーワードで検索した新規ユーザー
   → 悩み解決記事（30本＝スポーク／articles/）
   → note 使い方記事（ハブ／https://note.com/nikujuku/n/ne1774a6d11a3 ）
   → アプリ（https://nekko-weld.vercel.app ）
```

- **各スポーク記事の末尾CTAは、note使い方記事（ハブ）へのリンクに統一**しています（アプリURL直リンクは主CTAから外し、ハブ側でアプリへ変換）。
- **ハブ記事の完成版（そのまま貼れる全文リライト）**：[`note-hub-article.md`](./note-hub-article.md)。温かいトーン・体験談・失敗談入り、CTA3か所、呼称「戦法メモアプリ」で統一。既存note記事を差し替える想定。
- 既存note記事を部分改善したい場合の差し込みテキスト集：[`note-hub-improvements.md`](./note-hub-improvements.md)（CTA強化・タイトル案・相互リンク案）。
- **呼称**：全体を「**戦法メモアプリ「ねっこ」**」に統一済み（記事08・15の「研究ノート／戦法ノート」は検索キーワードとしてあえて残置）。

## 記事の共通構成

各記事は次の型で統一しています（note／ブログにそのまま貼れます）。

1. **H1タイトル**：狙うキーワードを自然に含める（できるだけ前方に）
2. **導入**：読者の悩みに共感（2〜3文）
3. **原因**：なぜそうなるのかを、責めずに整理
4. **解決の考え方**：記事テーマの実用的な芯（ここが価値）
5. **ねっこの紹介**：テーマに最も関係する機能を1〜2個だけ自然に
6. **CTA**：note使い方記事（ハブ）へ誘導 → `https://note.com/nikujuku/n/ne1774a6d11a3`（ハブ側でアプリへ変換）
7. **フッター**：関連記事への内部リンク（回遊・SEO用）

## 運用メモ

- **公開先**：note／個人ブログを想定。note に貼る場合、H1は記事タイトル欄へ、本文の見出しはそのまま流用可。
- **meta description（ブログの場合）**：狙うキーワードを前方に置き、共感フックを添えて全角60〜80字程度。例）「将棋の戦法が覚えられないのは記憶力のせいではありません。丸暗記をやめ『使いどころ』で覚えるコツを紹介します。」
- **内部リンク**：各記事フッターの「あわせて読みたい」で相互リンク済み。ブログ移植時は相対パスを公開URLに置換する。
- **主CTA（導線）**：各記事の末尾CTAは note使い方記事（ハブ）へ集約。アプリへはハブ経由で誘導する。ハブ側の改善は [`note-hub-improvements.md`](./note-hub-improvements.md) を参照。
- **重複回避**：本文の「ねっこの紹介」で推す機能を記事ごとに変えて、毎回同じ売り込みにならないようにしている（管理表の「推す機能」列を参照）。
- **表記の一貫性**：戦法・囲い名はアプリ内 `src/data.js` の `STRATEGY_GROUPS`（居飛車／振り飛車／囲い）の表記に合わせている。
