# ねっこ 用語集

「ねっこ」は将棋の戦法・研究を**ツリー構造**で整理する研究ノートアプリ。
画面上の表記・コード上の変数名・DBのカラム名がそれぞれ異なる場合があるため、対応関係を整理する。

---

## 1. 全体構造

| 用語 | 説明 |
| --- | --- |
| **ツリー（Tree）** | 1つの戦法研究全体の単位。「中飛車」「角換わり」など戦法ごとに作成する。複数のノードで構成される。 |
| **ノード（Node）** | ツリーを構成する個々の局面・分岐点。「指し手の分岐」を表す。 |
| **ルートノード（Root Node）** | ツリーの一番根本にあるノード。ツリー作成時に自動で1つ作られ、通常はツリー名（戦法名）と同じラベルを持つ。マインドマップ上では「おおもとの戦法」というサブラベルが付く。 |
| **親ノード（Parent）** | あるノードから見て、1つ上位（根に近い側）のノード。 |
| **子ノード／分岐（Children / 分岐）** | あるノードから見て、1つ下位（葉に近い側）のノード群。画面上は「分岐」と表示される。 |
| **合流（Merge）** | 複数の親ノードから1つの子ノードへ線を引く特殊な関係。通常の親子関係（1対1）とは別に、追加で「合流元の親」を持たせられる。 |

### コード上の対応

| 画面表示 | フロント側プロパティ（JS） | DBカラム名 |
| --- | --- | --- |
| 親ノード | `parentId` | `parent_id` |
| 分岐（子ノード一覧） | `childIds`（DBには無く、`parent_id` から組み立てて算出） | ― |
| 合流元（追加の親） | `mergeParentIds` | `merge_parent_ids` |
| 合流対象フラグ | `isMergeTarget` | `is_merge_target` |
| ルートノード判定 | `isRoot` | `is_root` |

## 2. ノードが持つ情報（属性）

| 画面表示名 | フロント側プロパティ | DBカラム名 | 説明 |
| --- | --- | --- | --- |
| **ノード名** | `label` | `label` | ノードのタイトル（例：「▲４六銀型」）。 |
| **ステータス** | `status` | `status` | `done`（完成）／`wip`（研究中）／`todo`（未定）の3種類。 |
| **切り口**（コード上は「アプローチ」） | `approachType` | `approach_type` | この分岐が何によって枝分かれするかを表す分類。詳細は[3章](#3-切り口アプローチの種類)。 |
| **盤面** | `board` | `board` | 将棋盤の駒配置データ（9×9マス）。表示／非表示を切り替え可能で、非表示時は親ノードの盤面を引き継ぐ仕組み。 |
| **持ち駒**（先手／後手） | `handSente` / `handGote` | `hand_sente` / `hand_gote` | 盤面に紐づく持ち駒の数。 |
| **スタンプ** | `stamps` | `stamps` | 盤面のマスに置く印（強調マーカー）。 |
| **棋譜** | `kifu` | `kifu` | 盤面の変化を記録した手順のスナップショット配列。再生（ナビゲーション）して手順を確認できる。 |
| **メモ** | `memo` | `memo` | 気づき・方針・手順のポイントなどの自由記述。 |
| **並び順** | `sortOrder` | `sort_order` | 兄弟ノード間の表示順。 |

---

## 3. 切り口（アプローチ）の種類

ノード追加時に「何によってこの分岐が分かれるか」を選ぶ分類。コード上は `approachType` / `APPROACH_META` という名前。

| 切り口 | 意味 | マインドマップでの色分け |
| --- | --- | --- |
| **自分の戦法** | 自分が指す戦法による分岐（例：四間飛車／角換わり／矢倉） | 青系（自分の手） |
| **相手の戦法** | 相手の出方による分岐（例：居飛車／三間飛車／穴熊） | 赤茶系（相手の手）・破線 |
| **局面の状況** | 盤面の条件による分岐（例：銀が間に合った／穴熊に組まれた） | 茶色・破線 |

> 補足：「自分の志向」（攻め重視／守り重視 など）は「自分の戦法」を選んだ際の付加情報（タグ的な入力欄）として登場するが、上記3種の切り口とは別枠の概念。

---

## 4. ステータスの種類

ノード・ツリーカードの進捗を表す。コード上は `STATUS_META`。

| 表示ラベル | 値（コード上） | 色 | 意味 |
| --- | --- | --- | --- |
| 完成 | `done` | 緑 | 研究が完了している分岐。 |
| 研究中 | `wip` | オレンジ | 検討中・未完成の分岐。 |
| 未定 | `todo` | グレー（破線） | まだ手を付けていない分岐。 |

---

## 5. ツリーが持つ情報（属性）

| 画面表示名 | DBカラム名 | 説明 |
| --- | --- | --- |
| **戦法名** | `name` | ツリーの名前（例：「中飛車」）。新規作成・編集モーダルでは「戦法名」というラベルで入力する。 |
| **タグ** | `tags` | カンマ・読点・スペース区切りで複数登録できる分類用の語句（例：「振り飛車」「中飛車」）。検索・絞り込みにも使う。 |
| **使用中／休止中** | `active` | そのツリー（戦法）を現在実戦で使っているかどうかのステータス。一覧画面でセクション分けされる。 |
| **公開中**（非公開） | `is_public` | 「みんなのツリー」に公開しているかどうか。 |
| **いいね数** | `liked_by` | 公開ツリーに対する「いいね」の数。 |
| **一言メモ** | `quick_memo` | ツリー一覧カードに付けられる、ノードとは無関係な簡易メモ。 |

## 6. 画面・機能名

| 画面表示名 | コンポーネント名 | 説明 |
| --- | --- | --- |
| （ツリー一覧画面） | `TreeList` / `TreeCard` | 自分のツリー一覧。「使用中」「休止中」でグループ表示。 |
| （マインドマップ画面） | `MindMap` | ツリーをSVGの樹形図として表示。ドラッグで親子付け替え（合流設定はここでは不可）、ピンチ／ホイールで拡大縮小。 |
| **目次** | `Accordion`（ドロワー） | マインドマップ右側から開く、ノード一覧のアコーディオン表示。番号（1, 1-1, 1-2…）付きでノードへジャンプできる。 |
| （ノード詳細画面） | `NodeDetail` | ノード名・切り口・ステータス・メモ・盤面・親ノード・分岐（子ノード）・合流設定・削除を扱う編集画面。 |
| **分岐を追加 / 詳細を入力**（ノード追加ウィザード） | `NewNode` | 新規ノード作成の2ステップウィザード。STEP0で「切り口」を選び、STEP1で「ノード名」「メモ」等を入力。STEP1冒頭の「分岐元：」は追加元の親ノード名を表す。 |
| **みんなのツリー** | `PublicTrees` / `PublicTreeCard` | 他ユーザーが公開しているツリーの一覧。検索・タグ絞り込み・コピー（自分のツリーとして複製）が可能。 |
| **トロフィー** | `TrophyScreen` | 累計のツリー数・ノード数・ログイン日数・連続ログイン日数とバッジ（実績）の獲得状況を表示。 |

## 7. 将棋盤・棋譜まわりの用語（`ShogiBoard.jsx`）

| 用語 | 説明 |
| --- | --- |
| **盤面** | 9×9の駒配置データ。各ノードが個別に持てる（持たない場合は親の盤面を継承表示）。 |
| **持ち駒**（自分の持ち駒／相手の持ち駒） | 先手（`handSente`）・後手（`handGote`）それぞれの手駒の数。 |
| **成り（成る）** | 駒が敵陣に入った際の昇格。`PROMOTABLE` / `PROMOTED_LABEL` で定義。 |
| **スタンプ** | 盤面のマスに重ねて表示する強調マーカー（注目マスを示す）。 |
| **棋譜（記録／再生）** | 盤面操作を1手ごとのスナップショットとして記録し、後から手順を再生（前後ナビゲーション）できる機能。「棋譜 N手」のように手数で表示。 |
| **編集ツール（動かす／スタンプ／消す）** | 盤面編集モードの切り替えタブ。 |

---

## 8. ご褒美（バッジ／トロフィー）関連（`rewards.js`）

| 用語 | 説明 |
| --- | --- |
| **バッジ** | ツリー数・ノード数・連続ログイン日数などの条件を満たすと獲得できる実績アイコン（`BADGE_DEFS`）。 |
| **累計ログイン** | これまでにログインした日数の合計。 |
| **連続ログイン** | 現在何日連続でログインしているか。 |

---

## 9. ユーザー・認証関連

| 用語 | 説明 |
| --- | --- |
| **ID** | ログイン用のユーザー名（内部的には `username@nekko.local` という仮メールアドレスに変換してSupabase Authに登録）。 |
| **表示名（display_name）** | アプリ内の他ユーザーに表示される名前。未設定時はIDが使われる。 |
| **プロフィール（profile）** | `profiles` テーブルに保存されるユーザー情報（`username`, `display_name` など）。 |

---

## 10. 用語の階層イメージ

```
ツリー（戦法名・タグ・使用中/休止中・公開設定・一言メモ）
└─ ルートノード（おおもとの戦法）
    └─ ノード（ノード名・切り口・ステータス・盤面・持ち駒・スタンプ・棋譜・メモ）
        ├─ 親ノード（1つ）
        ├─ 分岐＝子ノード（複数）
        └─ 合流元／合流先（追加の親子関係、複数可）
```
